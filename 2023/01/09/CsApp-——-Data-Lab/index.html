这题主要考察如何实现大小比较。通过两数相减的结果的正负来判断两个数之间的大小关系，但是因为不支持减法，所以需要进行转换。a-b 即 a + (-b)，而-b又可以表示为 ~b +1。
因此一般的思路即将x >= 0x30 和 x <= 0x39的结果去一个交集。x >= 0x30通过greater而得，因为符号位为0，取反即为1。而less，需要得负值或0，因此除了其他小于0的情况，还得将0x39考虑进来，因此就有了!less。

本题还是和上一题一样，先求出浮点数的各个参数。然后将exp换算成e。frac需要加上缺省的1，变成M。
而e即代表着2的e次方，因此当e > 31时，数值就可能溢出；当e < 0时，即数值为小数，因此取整后结果为0。
剩下e 处于[0,31],由于frac的位数为23，因此23又是e的一个临界值，当e > 23时，说明frac当前的位数不够，还需将结果继续左移 (e - 23)位；当 e < 23时，说明frac的结果需要右移几位，才能将结果用frac进行表达。
等讨论完e和frac的情况后，再来看符号，因为frac不带符号，所以需要根据符号做出调整。符号为负数时，需要将frac取反，因为不能使用-，则使用~frac + 1；符号为正，则直接返回。
